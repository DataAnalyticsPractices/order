# 订单管理系统

一个基于 Vite + Vue 3 + Express + MongoDB 的本地订单管理系统,用于管理生产账单和发货货款。

## 功能特点

### 1. 生产账单信息
- ✅ 创建和管理生产订单(工厂名称、货物信息、数量、单价、生产总价)
- ✅ 添加材料/其他费用记录(支持多种类型:面料、五金、复合、织袋、塑料袋、里布、其他)
- ✅ 添加付款记录(支付给工厂或材料厂)
- ✅ 自动计算应付金额(生产总价 - 材料费用 - 已付款)
- ✅ 多条件筛选(工厂名称、材料类型、关键词、时间段)
- ✅ 实时显示总欠款
- ✅ 导出Excel(全部数据和筛选结果两种方式)

### 2. 发货货款统计
- ✅ 创建和管理发货订单(订货方、货物图片、货物名称/货号、编号、日期等)
- ✅ 添加付款记录(定金、已付金额,支持负数)
- ✅ 自动计算欠款金额
- ✅ 多条件筛选(订货方、货品名称/货号、付款状态、关键词、时间段)
- ✅ 实时显示客户总欠款
- ✅ 导出Excel

## 技术栈

- **前端**: Vue 3 + Vite + Element Plus
- **后端**: Node.js + Express
- **数据库**: MongoDB
- **其他**: Axios、XLSX、File-Saver

## 安装步骤

### 前置要求

1. 安装 Node.js (版本 >= 18)
2. 安装 MongoDB Community Edition

### 1. 安装 MongoDB

#### Windows:
1. 下载 MongoDB Community Server: https://www.mongodb.com/try/download/community
2. 运行安装程序,选择"Complete"安装
3. 创建数据目录: `mkdir C:\data\db`
4. 启动 MongoDB 服务:
   ```powershell
   # 以管理员身份运行
   net start MongoDB
   ```

   或者直接运行:
   ```powershell
   "C:\Program Files\MongoDB\Server\7.0\bin\mongod.exe" --dbpath="C:\data\db"
   ```

### 2. 安装项目依赖

在项目根目录打开终端,运行:

```bash
npm install
```

### 3. 启动应用

#### 方式一: 同时启动前后端(推荐)
```bash
npm start
```

#### 方式二: 分别启动

**终端 1 - 启动后端服务器:**
```bash
npm run server
```

**终端 2 - 启动前端开发服务器:**
```bash
npm run dev
```

### 4. 访问应用

打开浏览器访问: http://localhost:3000

### 5. 导入示例数据 (可选)

如果想要快速体验系统功能,可以导入示例数据:

```bash
npm run seed
```

这会在数据库中创建一些示例订单,包括:
- 3 个生产订单(含材料和付款记录)
- 4 个发货订单(含付款记录)

**注意:** 运行此命令会清空现有数据!

## 使用说明

### 生产账单信息

1. **新建订单**: 点击"新建订单"按钮,填写工厂名称、货物信息、数量等
2. **添加材料**: 展开订单,点击"添加材料"按钮,选择材料类型、填写价格等
3. **添加付款**: 展开订单,点击"添加付款"按钮,记录付款信息
4. **筛选查询**: 使用顶部筛选条件查找特定订单
5. **导出数据**: 
   - "导出全部" - 导出所有订单数据
   - "导出筛选结果" - 只导出当前筛选的数据

### 发货货款统计

1. **新建订单**: 点击"新建订单"按钮,填写订货方、货物信息等
2. **上传图片**: 在货物图片字段粘贴图片URL或base64编码
3. **添加付款**: 展开订单,点击"添加付款"按钮,记录定金和已付金额
4. **筛选查询**: 使用顶部筛选条件查找特定订单
5. **导出数据**: 点击"导出Excel"按钮导出当前数据

## Excel导出说明

### 生产账单导出内容
- **订单汇总**: 所有订单的基本信息和金额汇总
- **详细信息**: 每个订单的完整信息,包括材料记录和付款记录
- **材料汇总**: 按材料类型和备注统计的汇总表

### 发货货款导出内容
- **订单汇总**: 所有订单的基本信息和金额汇总
- **详细信息**: 每个订单的完整信息,包括付款记录
- **客户汇总**: 按客户统计的订单数量和金额

## 数据库

- 数据库名称: `order_management`
- 集合:
  - `productionorders` - 生产订单
  - `shippingorders` - 发货订单

## 注意事项

1. 首次运行前确保 MongoDB 服务已启动
2. 价格可以输入负数(用于记录退款等情况)
3. 系统会自动计算应付金额和欠款
4. 所有数据存储在本地 MongoDB 数据库中
5. 建议定期备份数据库

## 常见问题

### MongoDB 连接失败
- 确认 MongoDB 服务已启动
- 检查端口 27017 是否被占用
- 查看防火墙设置

### 前端无法连接后端
- 确认后端服务器已启动(端口 5000)
- 检查控制台错误信息
